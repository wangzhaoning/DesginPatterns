单件模式和创建全局变量的区别
创建全局变量必须在程序的开始就创建好，但是很多时候我们往往在一次执行过程中没用到它，这就形成了浪费。单件模式可以在想使用它的时候在创建。

单件模式：确保一个类只有一个实例，并提供一个全局访问点。


这样当两个线程时候可能会造成冲突。解决方法之一是给静态方法加上同步锁，但是如此性能会降低100倍；


单例模式的优点：减少了内存的开支，性能的开销，避免系统的多重占用。
缺点：扩展接口很困难，一般是无法扩展接口的。
使用场景：生成唯一序列号的时候
需要一个共享服务点，例如访问计数器
创建一个对象需要的资源过多的情况
需要定义打来嗯的静态常量和静态方法

设置有上限的多例模式

在spring中每个Bean默认都是单例的，这样做的优点是Spring容器可以管理这些bean的生命周期。
